This file provides guidance to AI agents (Claude Code, Copilot, etc.) working with this repository.

## Project Overview

**A2A Debugger** is a CLI tool for debugging and monitoring A2A (Agent-to-Agent) servers. It is part of the [Inference Gateway](https://github.com/inference-gateway) ecosystem.

| Technology | Purpose |
|---|---|
| **Go 1.26.2** | Core language — module path `github.com/inference-gateway/a2a-debugger` |

| **[Viper](https://github.com/spf13/viper)** (`v1.21.0`) | Configuration management (file + env + flags) |
| **[inference-gateway/adk](https://github.com/inference-gateway/adk)** (`v0.17.3`) | A2A client library (`client.A2AClient` interface + `types` package) |
| **[zap](https://go.uber.org/zap)** (`v1.28.0`) | Structured logging |
| **[Task](https://taskfile.dev/)** | Build system runner |
| **GitHub Actions** | CI, Release, Artifact management |
| **Docker / GoReleaser** | Container images and binary distribution |
| **[semantic-release](https://semantic-release.gitbook.io/)** | Automated versioning and changelog |

### Key Facts

- **Binary name**: `a2a` (installed to `$HOME/.local/bin` by default)

## Architecture

### Entry Point (`main.go`)

```go
func main() {
    cli.Execute(version, commit, date)
}
```

Version info (`version`, `commit`, `date`) is injected via `ldflags` at build time. The `main.go` delegates everything to `cli.Execute()`.

### CLI Layer (`cli/cli.go`)

All commands live in a **single file** (`cli/cli.go`). There are no sub-packages. The architecture follows these patterns:

#### Command Namespace Structure

```
a2a
├── config              # Configuration management
│   ├── set <key> <val>
│   ├── get <key>
│   └── list
├── tasks               # Task operations
│   ├── list
│   ├── get <task-id>
│   ├── history <context-id>
│   ├── submit <message>
│   └── submit-streaming <message>
├── connect             # Test server connection
├── agent-card          # Get agent card
└── version             # Print version info
```

#### Core Architectural Patterns

1. **`ensureA2AClient()`** — Lazy initialization of the A2A client. Must be called before any A2A RPC call. Initializes from `viper` config (`server-url`, `timeout`).

2. **`handleA2AError(err, method)`** — Centralized error handling. Detects `MethodNotFoundError` (JSON-RPC code `-32601`) and returns user-friendly messages.

3. **`formatOutput(data)` / `printFormatted(data)`** — Output formatting that supports YAML (default) and JSON, controlled by the `--output` / `-o` flag or `output` config key.

4. **`initConfig()`** — Viper init (runs via `cobra.OnInitialize`). Loads `~/.a2a.yaml` or custom config path. Environment variables are loaded automatically.

5. **`initLogger()`** — Creates a zap logger. Uses `zap.NewDevelopment()` when `--debug` is set, otherwise `zap.NewProduction()`.

#### A2A Client Interface

The A2A ADK (`github.com/inference-gateway/adk/client`) provides the `A2AClient` interface with these methods:
- `GetAgentCard(ctx)` — Returns `*adk.AgentCard`
- `GetHealth(ctx)` — Returns health info
- `ListTasks(ctx, params)` — Returns `*adk.JSONRPCSuccessResponse`
- `GetTask(ctx, params)` — Returns `*adk.JSONRPCSuccessResponse`
- `SendTask(ctx, params)` — Returns `*adk.JSONRPCSuccessResponse`
- `SendTaskStreaming(ctx, params)` — Returns `<-chan adk.JSONRPCSuccessResponse`
- `CancelTask(ctx, params)` — Cancels a task

Types are in `github.com/inference-gateway/adk/types` (e.g., `adk.Task`, `adk.TaskState`, `adk.Message`, `adk.Part`, `adk.AgentCard`). These are generated upstream from the A2A JSON-RPC schema — **do not modify them locally**. Bump the `adk` dependency to pick up schema changes.

## Testing Instructions

### Unit Tests

Tests use a **mock A2A client** (`mockA2AClient`) defined in `cli/cli_test.go`. The mock implements the full `A2AClient` interface and allows injecting canned streaming responses.

```go
// Pattern for testing streaming commands:
mockClient := &mockA2AClient{
    sendTaskStreamingFunc: func(ctx context.Context, params adk.MessageSendParams) (<-chan adk.JSONRPCSuccessResponse, error) {
        ch := make(chan adk.JSONRPCSuccessResponse, N)
        ch <- adk.JSONRPCSuccessResponse{Result: ...}
        close(ch)
        return ch, nil
    },
}
a2aClient = mockClient

// Capture stdout to verify output
oldStdout := os.Stdout
r, w, _ := os.Pipe()
os.Stdout = w
// ... run command ...
_ = w.Close()
os.Stdout = oldStdout
// Read captured output and assert
```

**Test files:**
- `cli/cli_test.go` — Tests for `submit-streaming` (streaming summary, raw mode, task snapshots)
- `cli/cli_output_test.go` — Tests for `getOutputFormat`, `formatOutput`, `printFormatted`

**Test patterns to follow:**
1. Save and restore global `a2aClient` and `logger` state
2. Replace `a2aClient` with mock before test, restore in defer
3. Capture `os.Stdout` to verify CLI output
4. Use `strings.Contains()` for output assertions

### End-to-End Testing

The `example/` directory provides a canonical E2E harness using Docker Compose:

```bash
cd example
docker compose up -d                            # Start mock A2A server
docker compose run --rm a2a-debugger connect                         # Test connection
docker compose run --rm a2a-debugger tasks list                      # List tasks
docker compose run --rm a2a-debugger tasks submit "Hello"            # Submit task
docker compose run --rm a2a-debugger tasks submit-streaming "Hi"     # Streaming task
```

This uses `ghcr.io/inference-gateway/mock-agent:latest` — a mock A2A server that requires **no API keys**.

## Adding New Commands

When adding new commands to `cli/cli.go`:

1. **Define the command** using `&cobra.Command{...}`
2. **Use `ensureA2AClient()`** before making any A2A RPC calls
3. **Use `handleA2AError(err, method)`** for error handling
4. **Use `printFormatted(data)`** for structured output (respects `--output` format)
5. **Register** in the `init()` function (add to parent command with `AddCommand`)
6. **Add flags** — use global flags on `rootCmd.PersistentFlags()` or command-specific on the command itself
7. **Add tests** in `cli_test.go` or `cli_output_test.go`
8. **Document** the new command in `README.md`

### Command Implementation Pattern

```go
var myNewCmd = &cobra.Command{
    Use:   "my-command [arg]",
    Short: "Brief description",
    Long:  "Detailed description of what this command does.",
    Args:  cobra.ExactArgs(1),
    RunE: func(cmd *cobra.Command, args []string) error {
        ctx := context.Background()
        ensureA2AClient()

        // Business logic here
        result, err := a2aClient.SomeMethod(ctx, params)
        if err != nil {
            return handleA2AError(err, "my-command")
        }

        return printFormatted(result)
    },
}

func init() {
    // Register in appropriate parent
    parentCmd.AddCommand(myNewCmd)
    myNewCmd.Flags().String("my-flag", "default", "Description")
}
```

## Project Conventions

- **Go**: Standard `gofmt` / `goimports` formatting
- **Indentation**: tabs for `.go`, 2-space for `.yaml`/`.yml`/`.md` (enforced by `.editorconfig`)
- **Line endings**: LF (Unix), enforced by `.editorconfig`
- **Final newline**: Required on all files
- **Line length**: Max 120 for Go code
- **Import sorting**: Standard library, third-party, local (grouped with blank lines)
- **Naming**: Use `camelCase` for variables, `PascalCase` for exported symbols

### Commit Convention

This project uses **Conventional Commits** for automated changelog generation via semantic-release:

| Type | Description | Release |
|---|---|---|
| `feat` | New feature | minor |
| `fix` | Bug fix | patch |
| `impr` | Improvement | patch |

| `perf` | Performance | patch |
| `docs` | Documentation | patch |
| `ci` | CI changes | patch |
| `test` | Testing | patch |
| `chore` | Maintenance | patch |

Examples:
```
feat(tasks): Add --include-artifacts flag to tasks list command

```

### Git Workflow

- **Branch from**: `main`
- **Branch prefix**: `claude/` (when using Claude Code action)
- **PR target**: `main`
- **Protected**: `main` branch — no direct pushes

### Testing Conventions

- Tests use **package-level** mock injection (swap global `a2aClient`)
- Always **restore globals** after test (`originalClient := a2aClient; defer func() { a2aClient = originalClient }()`)
- Use **table-driven tests** with `t.Run()` for multiple cases
- Capture `os.Stdout` via `os.Pipe()` for CLI output verification
- Use `strings.Contains()` for output assertions (not exact matching)

### Error Handling

- Always use `handleA2AError()` wrapper for A2A client errors
- The handler detects `MethodNotFoundError` (code `-32601`) and returns user-friendly messages with ❌ prefix
- Return structured errors from `RunE` functions (Cobra handles display)
- Use `fmt.Errorf("context: %w", err)` for error wrapping

## Important Files and Configurations

### Configuration (`~/.a2a.yaml`)

```yaml
server-url: http://localhost:8080
timeout: 30s
debug: false
insecure: false
output: yaml  # or json
```

All config keys can be set via:
1. Config file (`~/.a2a.yaml` or `--config`)
2. CLI flags (`--server-url`, `--timeout`, etc.)
3. Environment variables (Viper `AutomaticEnv` — maps to `A2A_*` env vars)

Default values:
- `server-url`: `http://localhost:8080`
- `timeout`: `30s`
- `debug`: `false`
- `insecure`: `false`
- `output`: `yaml`

### Build (`Taskfile.yaml`)

The `Taskfile.yaml` defines these tasks:
- `default` — List available tasks
- `tidy` — `go mod tidy`
- `lint` — `golangci-lint run`
- `build` — Build with ldflags version injection
- `build:dev` — Build without version info
- `build:docker` — Docker build with version tags
- `install` / `uninstall` — Local install/uninstall
- `test` / `test:coverage` — Run tests
- `clean` — Remove build artifacts

### Docker

**`Dockerfile`** — Multi-stage build (for dev/manual builds):
1. Stage 1: `golang:1.26.2-alpine` builder — compiles binary
2. Stage 2: `alpine:latest` — minimal runtime with CA certs
3. Entrypoint: `/a2a`

**`Dockerfile.goreleaser`** — For GoReleaser automation:
1. Stage 1: `alpine:latest` — compresses binary with UPX
2. Stage 2: `gcr.io/distroless/static-debian12:nonroot` — minimal, non-root
3. User: `nonroot:nonroot`

### CI/CD Pipeline

| Workflow | Trigger | Actions |
|---|---|---|
| `ci.yml` | Push/PR to `main` | tidy → dirty-check → lint → build → test |
| `release.yml` | Manual (`workflow_dispatch`) | semantic-release → version bump → CHANGELOG → git tag |
| `artifacts.yml` | Release published + manual | GoReleaser → binaries → Docker images → Trivy scan → Cosign sign |
| `claude.yml` | Issue/PR comments with `@claude` | Claude Code action for automated fixes |

### Release Process

Managed by semantic-release (`.releaserc.yaml`):
1. Trigger: manual `workflow_dispatch` on `release.yml`
2. Analyzes commits since last release using conventional commits
3. Determines next version (major/minor/patch/rc)
4. Updates `CHANGELOG.md`
5. Creates git tag
6. Triggers `artifacts.yml` (release published) which:
   - Builds binaries (Linux amd64, arm64, arm7 via GoReleaser)
   - Builds & pushes Docker images to `ghcr.io/inference-gateway/a2a-debugger`
   - Scans containers with Trivy
   - Signs images with Cosign

### A2A ADK Dependency

- **Source**: `github.com/inference-gateway/adk`
- **Package**: `client` — A2A client interface and implementation
- **Package**: `types` — All A2A protocol types (task, message, part, agent card, etc.)
- **Updating**: Bump the version in `go.mod` — **no local code generation**
- **Types are generated** upstream from the A2A JSON-RPC schema

### Install Script (`install.sh`)

Supports Linux and macOS (x86_64, arm64). Downloads pre-built binaries from GitHub releases. Options:
```bash
curl -fsSL https://raw.githubusercontent.com/inference-gateway/a2a-debugger/main/install.sh | bash
./install.sh --version v1.0.0    # Specific version
INSTALL_DIR=~/bin ./install.sh   # Custom path
```

## Common Tasks for AI Agents

### I need to fix a bug
1. Find the relevant command in `cli/cli.go`
2. Identify if it's an A2A client error → use `handleA2AError()`
3. Add/update tests in the relevant `*_test.go` file
4. Run `task test` to verify
5. Commit with `fix(scope): Description`

### I need to add a new feature
1. Add the new `cobra.Command` in `cli/cli.go`
2. Register in `init()`
3. Add tests in `cli_test.go` or `cli_output_test.go`
4. Document in `README.md`
5. Run `task lint && task build && task test`
6. Commit with `feat(scope): Description`

### I need to update A2A types
1. Update the `adk` dependency in `go.mod`
2. Run `task tidy`
3. Fix any compilation errors in `cli/cli.go`
4. Run `task test`

### I need to debug streaming issues
1. Use `--raw` flag: `a2a tasks submit-streaming "message" --raw`
2. This shows raw JSON-RPC events instead of formatted output
3. Use the example docker-compose: `cd example && docker compose up -d`
4. Run streaming against the mock agent
